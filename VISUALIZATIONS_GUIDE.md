# Visualization Guide

Guide to the presentation figures generated by step 08.

---

## Overview

Step 08 creates 6 publication-quality figures for presentations and reports. All figures are:
- High resolution (300 DPI)
- Sized for slides (10"×6")
- Styled with clear fonts and high-contrast colors
- Saved as PNG files in `data/figures/`

---

## Figure Descriptions

### 1. Ancestry Distribution (`01_ancestry_distribution.png`)

**What it shows:** Bar chart of sample counts across 5 ancestry groups (tier0).

**Key features:**
- Color-coded by ancestry group (defined in config.py)
- Value labels on each bar
- Gridlines for easy reading

**Use in presentation:**
- Introduce your dataset demographics
- Show diversity of samples
- Justify ancestry-stratified analyses

**Interpretation:**
- EUR (European): Largest group
- Admixed: Mixed ancestry
- Founder: Jewish populations
- Non-EUR: Asian, African, etc.
- Unknown: Unable to classify

---

### 2. QC Waterfall (`02_qc_waterfall.png`)

**What it shows:** Dual bar charts showing sample and SNP counts through QC filters.

**Key features:**
- Left panel: Samples
- Right panel: SNPs
- Shows reduction at each QC step
- Annotations show how many removed

**Use in presentation:**
- Demonstrate rigorous quality control
- Show filtering criteria were applied
- Justify final sample size

**Interpretation:**
- Each bar = one QC stage
- Arrows and numbers show reductions
- Final bars = dataset for analysis

---

### 3. IBS Distribution (`03_ibs_distribution.png`)

**What it shows:** Two histograms showing genetic similarity across all sample pairs.

**Key features:**
- Left: IBS2 proportion (fraction of loci where both alleles match)
- Right: PI_HAT (estimated IBD sharing)
- Vertical lines show relatedness thresholds
- Statistics boxes with key numbers

**Use in presentation:**
- Main results figure
- Show genetic similarity patterns
- Identify related individuals

**Interpretation:**
- Most pairs have low similarity (unrelated)
- Right tail = related pairs
- Thresholds define relationship degrees
- Median values indicate population structure

---

### 4. Relatedness by Ancestry (`04_relatedness_by_ancestry.png`)

**What it shows:** Analysis of related pairs (PI_HAT > 0.125).

**Key features:**
- Left: Pie chart of ancestry concordance
- Right: Bar chart of relationship degrees
- Shows within vs. between ancestry relatedness

**Use in presentation:**
- Demonstrate cryptic relatedness
- Show ancestry patterns
- Identify potential duplicates or close relatives

**Interpretation:**
- Same ancestry: Related pairs from same group
- Different ancestry: Admixed or misclassified?
- 1st/2nd/3rd degree: Relationship closeness
- Duplicates: Same person, technical replicates, or MZ twins

---

### 5. Pipeline Overview (`05_pipeline_overview.png`)

**What it shows:** Line graph showing sample counts through entire pipeline.

**Key features:**
- X-axis: Pipeline stages
- Y-axis: Number of samples
- Arrows show reductions between stages
- Annotations explain losses

**Use in presentation:**
- Methods figure
- Show complete pipeline workflow
- Justify filtering decisions

**Interpretation:**
- Starts at 414 users
- Drops to 329 after data prep
- Further reduced through conversion/QC
- Final: 260 high-quality samples

---

### 6. Data Quality (`06_data_quality.png`)

**What it shows:** Three panels showing input data characteristics.

**Key features:**
- Left: File size distribution
- Middle: SNP count per file
- Right: Format distribution (pie chart)

**Use in presentation:**
- Characterize input data
- Show data heterogeneity
- Justify filtering approach

**Interpretation:**
- File sizes: Most 10-25 MB (typical for SNP arrays)
- SNP counts: Most 600k-900k SNPs
- Formats: Mix of 23andMe and Ancestry platforms

## Using in Presentations

### PowerPoint/Keynote

1. Insert figure: Insert → Picture → Choose PNG file
2. Resize if needed (maintain aspect ratio)
3. Add figure caption below
4. Reference in text as "Figure X"

### Suggested order

For a typical results presentation:

1. **Intro:** Pipeline Overview (show workflow)
2. **Methods:** Data Quality + Ancestry Distribution
3. **Results:** QC Waterfall → IBS Distribution → Relatedness by Ancestry
4. **Conclusion:** Highlight key findings from IBS/relatedness figures

### Tips

- Use full slide for each figure (better visibility)
- Add title slide before each figure
- Reference specific features in oral presentation
- Have high-res versions ready for print/posters

## Customizing Figures

### Change colors

Edit `config.py`:
```python
TIER0_COLORS = {
    "EUR": "#your_color_hex",
    # ... etc
}
```

### Adjust figure size

Edit `08_visualizations.py`:
```python
plt.rcParams['figure.figsize'] = (12, 8)  # width, height in inches
```

### Change DPI

Edit `08_visualizations.py`:
```python
plt.rcParams['figure.dpi'] = 600  # higher = better quality, larger files
```

### Modify plot types

Each plot has its own function in `08_visualizations.py`:
- `plot_ancestry_distribution()`
- `plot_qc_waterfall()`
- etc.

Edit these functions to customize appearance.

## Troubleshooting

**No figures generated:**
- Check step 07 completed successfully
- Verify output files exist: `ls data/plink_qc/`

**Figures look different:**
- matplotlib/seaborn versions may vary
- Update: `pip install --upgrade matplotlib seaborn`

**Can't open PNG files:**
- Use any image viewer
- macOS: `open data/figures/01_ancestry_distribution.png`
- Linux: `xdg-open data/figures/01_ancestry_distribution.png`

**Poor quality when printed:**
- Increase DPI in script (300 → 600)
- Or save as SVG for vector graphics
